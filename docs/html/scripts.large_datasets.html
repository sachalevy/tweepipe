scripts.large_datasets.reorder_hydration_keys module¶

scripts.large_datasets.reorder_hydration_keys.main()¶

When working on a large hydration database, a load balancing method is to pre-attribute a key index to each tweet such that the apis don’t overlap in the tweets they process. However, if a process fails, and is not handled, one of the keys may end up being left behind with a lot more tweets to retrieve than other keys. This program re-order the keys of the tweets left to retrieve.
